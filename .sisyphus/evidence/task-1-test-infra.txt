## Test Execution

dotnet test -p:EnableWindowsTargeting=true: CANNOT RUN ON LINUX
  Reason: net8.0-windows requires Microsoft.WindowsDesktop.App runtime (8.0.0)
  which is only available on Windows. Tests will pass on Windows machines.

  On Linux, the build succeeds but testhost cannot load WPF dependencies.
  This is an inherent limitation of WPF projects on non-Windows platforms.

## Key Learnings

1. WPF projects on Linux require EnableWindowsTargeting=true for compilation
2. WPF temporary project (_CompileTemporaryAssembly) includes all .cs files
   from subdirectories - must explicitly exclude test project directories
3. Caption namespace is LiveCaptionsTranslator.models (capital L, not lowercase)
4. net8.0-windows tests cannot execute on Linux (no WindowsDesktop runtime)
5. .NET 8 SDK was installed to ~/.dotnet (not pre-installed on this system)
